## 実装メモ

### 全般的な変更点

- 古い型参照 `@/types/supabase` を新しい型参照 `@/types/network/supabase` に変更
- 型安全性の強化: できるだけ `any` や `unknown` の型キャストを避け、適切な型を使用
- エラーハンドリングの強化: エラーメッセージの詳細化と一貫性の確保
- コメントの充実: メソッドの役割と使用方法の明確化

### 型定義の主な変更点

- `PublicSchema` → `DefaultSchema` に名称変更
- `Json`型の活用: ユーザー設定などの任意のJSON構造に対応

### Mem0-Supabase連携の実装

- Supabaseにmemoriesテーブルを作成
- pgvector拡張を使用したベクトル検索機能の実装
- SupabaseMem0Integrationクラスによる2重保存と冗長性
- APIエンドポイントの追加（`/api/memories`）
- ユーザーIDに基づくメモリ分離とアクセス制御
