# 未使用型の削除計画

## 概要

ts-pruneによる静的解析の結果、多数の未使用型が検出されました。これらを整理・削除することでコードベースをスリム化し、保守性を向上させます。

## 削除の基本方針

1. **段階的アプローチ**
   - ドメインごとに未使用型を分類
   - 優先度の高いものから段階的に削除
   - 各段階でビルドテストを実施

2. **安全性の確保**
   - 削除前に依存関係の確認
   - 直接参照されていなくても継承などで使用されている可能性を考慮
   - バレルファイル経由のエクスポートも考慮

3. **ドキュメント化**
   - 削除する型の一覧を記録
   - 削除の理由を明記

## 未使用型分類（ts-pruneの結果から）

### 優先度高（即時削除候補）
- 旧ファイルからの再エクスポート型
- ダミー・テスト用の型
- 明らかに使用されていない実験的な型

### 優先度中（検証後削除）
- store/barrel.tsの多数のセレクタ型
- 一部のツール・ユーティリティ関数の型
- 重複している可能性のある型

### 優先度低（保留）
- Next.js関連の自動生成型
- 将来的に使用する可能性のある型
- 外部ライブラリとの統合用の型

## 実施手順

1. **ドメインごとの分析**
   ```bash
   npx ts-prune | grep 'types/chart'
   npx ts-prune | grep 'types/store'
   # 以下同様に各ドメインごとに実行
   ```

2. **削除対象リストの作成**
   - 各ドメインごとに削除する型のリストを作成
   - コミットメッセージに含める説明文を準備

3. **段階的な削除作業**
   - 各ドメインについて以下を実行:
     1. 未使用型の削除
     2. バレルファイルの更新
     3. 型チェックとビルドテスト
     4. コミット

4. **最終確認**
   - すべての削除後に総合テスト
   - CI/CD環境での検証

## スケジュール

|  #  | 作業内容 | 目標 |
| --- | -------- | ---- |
| 1 | store/barrel.tsの未使用セレクタ削除 | セレクタの整理と重複排除 |
| 2 | types/validationsの未使用スキーマ削除 | バリデーション関連の型の整理 |
| 3 | types/networkの未使用型削除 | ネットワーク関連型の整理 |
| 4 | その他のドメイン別未使用型削除 | 残りの未使用型の整理 |
| 5 | 最終検証 | すべての機能が正常に動作することを確認 |
