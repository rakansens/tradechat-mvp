## 傾向と主な問題カテゴリ

1. **ExchangeType定義不一致** - 全エラーの約40%を占める最大の問題
   - `"spot" | "futures"` と `"bitget" | "binance" | "bybit" | "demo"` の不一致
   - 異なるファイルから異なる定義をインポートしている

2. **型定義のリファクタリング不全** - 約30%
   - storeのリファクタリングに伴い、古い参照パスや型定義が残存

3. **暗黙的any型** - 約10%
   - 特にコンポーネント等でパラメータの型が明示されていない

4. **テストファイルの型不整合** - 約15%
   - 本番コードの型変更に伴いテストコードが更新されていない

## 修正優先度

1. S-9: **ExchangeType統一** - 最大の問題源を解決
2. S-10: **Symbol/Chart系any撲滅** - 型安全性強化
3. S-11: **SliceCreator移行** - ストア構造の整合性確保
4. S-12: **テスト修正またはスタブ化** - CI緑化

## 注意事項

- 型定義の変更は慎重に実施し、既存の参照箇所をすべて確認する
- 段階的な修正を心がけ、各修正後にTSCでエラー減少を確認する
